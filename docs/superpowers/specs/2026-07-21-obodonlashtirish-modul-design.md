# Ободонлаштириш модули — дизайн лойиҳаси

> Манба: раҳбар шахсан чизган икки эскиз (маҳалла-ичи умумий тузилма + ободонлаштириш жадвали).
> Мослик: мавжуд Mahalla домени (`master.streets`, `master.buildings`, `street_assignments`,
> зона мониторинги, микролойиҳа) ва яшил тема билан тўлиқ уйғун.

**Мақсад:** Маҳалла бўйича ободонлаштириш ишларини **кўча кесимида, масъуллар бўйича**
кузатиш. Ҳар кўчада хонадонлар сони ва 4 иш тури (Фасад, Томирқа, Хона-таҳона,
Эҳота-дарвоза) бўйича **бажарилди/қолди** прогресси. Маълумотни масъуллар мобил
илова орқали киритади; раҳбарият (вилоят/туман) веб-панелда мониторинг қилади.

## 1. Иккита экран

### 1.1. Маҳалла кесими (1-эскиз) — `ExecutiveMahalla.vue` кенгайтирилади
Раҳбар маҳаллага кирганда кўрадиган саҳифа. Юқоридан пастга:
- **Аҳоли статистикаси** (карточкалар): Аҳоли сони, Хонадон сони, Оилалар сони, Майдони.
  - Манба: `master.buildings` (хонадон), `mahalla_indicators` (аҳоли/оила — база файлдан),
    `mahallas.boundary` (майдон, PostGIS `ST_Area`).
- **Асосий бўлимлар** (4 та катта карточка, босиладиган):
  1. **Ободонлаштириш** ← шу модул (тўлиқ)
  2. Солиқ тушуми (каркас — кейинги босқич)
  3. Микролойиҳалар (мавжуд: `micro_projects`)
  4. Ижтимоий объектлар (мавжуд: `buildings.object_type`)
- **Таҳлилий кўрсаткичлар** (кичик карточкалар, "?" — база/ташқи манба):
  Ишсизлар (норасмий), Корхоналар, Бандлик реестри, Олий таълим битирган ишсизлар,
  Олинган кредитлар. (Каркас — маълумот манбаси аниқлангач тўлдирилади.)
- **Рўйхат ва реестрлар** (чиплар): Ёшлар дафтари, Аёллар дафтари, ПК-1, Миграция,
  Ҳайвонот, Камбағал оилалар. (Каркас.)

### 1.2. Ободонлаштириш модули (2-эскиз) — янги саҳифа/таб
Маҳалла ичидаги "Ободонлаштириш" карточкасидан очилади.
- **Хулоса**: Жами хонадон, Умумий бажарилди %, ҳар иш тури бўйича мини-прогресс.
- **Матрица жадвали**: **Масъул → бириктирилган кўчалар → иш турлари**.
  - Гуруҳ: 5 масъул (Маҳалла раиси, Ҳоким ёрдамчиси, Ёшлар етакчиси,
    Хотин-қизлар фаоли, Ижтимоий ходим).
  - Устунлар: Кўча | Хонадон сони | Фасад (баж./қолди) | Томирқа (баж./қолди) |
    Хона-таҳона (баж./қолди) | Эҳота-дарвоза (баж./қолди) | Умумий %.
  - **Қолди = Хонадон сони − Бажарилди** (автоматик — фақат "бажарилди" сақланади).
  - Ҳар масъул остида жамланма қатор + маҳалла бўйича умумий "ЖАМИ" қатор.

## 2. МУҲИМ ТУЗАТИШ — маълумот сурат орқали, рақам эмас

Раҳбар изоҳи: «бажарилди» рақами **қўлда киритилмайди**. Мавжуд
**«Хонадон таъмир назорати» мобил илова**си (`D:\kadr\mahalla_mobile`) айнан шу —
масъул ҳар бинога бориб, ҳар зона (иш тури) бўйича **жонли сурат** олади (GPS
геофенса ≤75 м, галерея йўқ). Сурат AI/масъул ходим томонидан тасдиқлангач,
зона ҳолати `completed` бўлади (`house_zone_states.status`).

Демак Ободонлаштириш жадвалидаги рақамлар — **мавжуд zone-holatларнинг кўча
кесимида жамланмаси**:
- **Бажарилди** (иш тури бўйича, кўчада) = шу кўчадаги, зона ҳолати `completed`
  (ёки `good`) бўлган бинолар сони.
- **Хонадон сони** = кўчадаги кадастр бинолари (`buildings`, type=residential) сони.
- **Қолди = Хонадон − Бажарилди** (авто).

Янги CRUD жадвал ШАРТ ЭМАС — мавжуд `zone_observations` / `house_zone_states` /
`buildings.street_id` етарли. Фақат **агрегация (rollup) сервиси** ва
(ихтиёрий) кампания/зона конфигурацияси керак.

### 2.1. Зона рўйхати — ҲАЛ ҚИЛИНАДИГАН САВОЛ
Ҳозирги 4 зона (`MahallaZones::ZONES`): **facade** (Уй фасади), **kitchen** (Ошхона),
**toilet** (Ҳожатхона), **yard** (Томорқа). Эскиздаги ободонлаштириш турлари:
**Фасад, Томирқа, Хона-таҳона, Эҳота-дарвоза**.
- Мослик: `facade`=Фасад, `yard`(Томорқа)=Томирқа.
- Фарқ: **Хона-таҳона** ва **Эҳота-дарвоза** — янги; **Ошхона/Ҳожатхона** — эскизда йўқ.

**Вариантлар** (фойдаланувчи/раҳбар танлайди):
- **(а) Зоналарни ободонлаштиришга мослаш** — `MahallaZones::ZONES` ни
  `facade, tomirqa, xona_tahona, ehota_darvoza` га алмаштириш (таъмир назорати ҳам
  шу турларга ўтади). Энг содда, лекин ошхона/ҳожатхона кузатувини йўқотади.
- **(б) 6 зона** — мавжуд 4 гa `xona_tahona` + `ehota_darvoza` қўшиш. Илова ҳар
  бинода 6 зона кўрсатади; Ободонлаштириш жадвали улардан 4 тасини (facade, yard,
  xona_tahona, ehota_darvoza) олади.
- **(в) Кампания** — ҳар маҳалла/давр учун "кампания" (таъмир / ободонлаштириш),
  ҳар бирининг ўз зона тўплами. Энг мослашувчан, кўпроқ иш.

> `MahallaZones::ZONES` const → `master.zone_types` луғатига кўчирилса, (б)/(в)
> кодсиз, seed билан ечилади.

### 2.2. (агар "хонадон сони" қўлда тузатиш керак бўлса — ихтиёрий)
`mahalla.obod_street_targets` (street_id UNIQUE per mahalla, household_count,
updated_by). Дефолт кадастрдан; масъул тузатиши учун. Йўқ бўлса — тўғридан-тўғри
кадастр сони ишлатилади.

## 3. API (янги, `/api/mahalla/*`)

### Раҳбарият (веб, `mahalla.viewer` — фақат кўриш) — АГРЕГАЦИЯ
- `GET /api/mahalla/executive/mahallas/{mahalla}/obod`
  → `{ summary{household_total, by_type[{code,name_cyr,done,total,pct}], overall_pct},
       groups[ { responsible{name,position}, streets[ { street_id, name,
       household_count, works[ {code, name_cyr, done, remaining, pct} ], overall_pct } ],
       subtotal } ], totals }`
  Ички: ҳар кўча учун `buildings` (household) + `house_zone_states` (completed сони)
  бўйича `GROUP BY street, zone` агрегат. Масъул = `street_assignments`дан.

### Масъул (мобил) — МАВЖУД endpoint'лар, ЎЗГАРИШСИЗ
- `GET /api/mahalla/worklist` — ўз кўчаларидаги бинолар + зона ҳолатлари.
- `GET /api/mahalla/buildings/{building}` — бино + зоналар + кузатувлар.
- `POST /api/mahalla/buildings/{building}/observations` — зона сурати юклаш
  (`zone, images[], captured_lat/lng, gps_accuracy`). Геофенса + AI серверда.

Тенант/скоуп: маҳалла профилдан (`MahallaAccess::scopeFor`), реквестдан эмас.

## 4. Мобил — МАВЖУД илова қайта ишлатилади (янги экран шарт эмас)

`D:\kadr\mahalla_mobile` — Flutter + Riverpod + Dio. Оқим ЎЗГАРМАЙДИ:
`WorklistScreen` (бинолар) → `HouseScreen` (4 зона + «Сурат») → `CaptureScreen`
(жонли камера + GPS) → юклаш (+ офлайн навбат).

Ягона ўзгариш — **зона рўйхати** (2.1 танловга қараб): агар (б)/(в) танланса,
`Zones.labels` (Flutter) + `MahallaZones::ZONES` (backend) га `xona_tahona`,
`ehota_darvoza` қўшилади. AI промптлари (`ZonePrompts`) ҳам шу турларга мосланади.

Изоҳ: масъул ҳеч қандай рақам термайди — сурат = киритиш. «Бажарилди/қолди»
раҳбар панелда авто-ҳисобланади.

## 6. Босқичлар (тасдиқдан кейин)
0. **Зона танлови** (2.1: а/б/в) — раҳбар билан тасдиқлаш. Керак бўлса зона
   рўйхатини кенгайтириш (backend `MahallaZones` + Flutter `Zones` + AI промпт).
1. Backend: **агрегация сервиси** (кўча×зона бўйича completed сони) +
   `GET /executive/mahallas/{mahalla}/obod` endpoint.
2. Frontend: ExecutiveMahalla'га "Асосий бўлимлар" + Ободонлаштириш таб/саҳифа
   (матрица жадвал — мавжуд `RaisOverview.streets` намунаси).
3. Мобил: зона рўйхати ўзгарса шуни янгилаш (акс ҳолда мавжуд оқим етарли).
4. (ихтиёрий) `obod_street_targets` — хонадон сонини қўлда тузатиш.
5. Локал тест → раҳбар тасдиғи → деплой (фойдаланувчи буйруғи билан).

> Ушбу ҳужжат — таклиф. Раҳбар визуал mockup'ни кўриб тасдиқлагач, имплементация бошланади.
